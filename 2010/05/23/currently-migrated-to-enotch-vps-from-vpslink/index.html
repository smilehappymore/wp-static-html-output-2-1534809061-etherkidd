Currently migrated to Enotch VPS from vpslink

May 23rd, 2010
Leave a comment | Trackback

due to numerous horrible stories from forum posts at forum.vpslink.com I have migrated to an enotch vps.

How do you like the performance?  It will remain here for at least a week while site monitors will check on vpslink reliability in the boston datacenter.

The process of my backup was a very simple one since my dns services is from a 3rd party (not using vpslink).

First thing is backup the directory of my http files.  This is a simple process if you have a consistant directory structure separated by domain folders. I used the simple command like this:

tar -czf vpslinkbackup.tar.gz *

Than I use a mysql dump of all databases which is fine because I have a database of each domain

mysqldump -u root -p<password> --all-databases > vpslinkdb.sql

Very simple.  I change my a records away from vpslink at my 3rd party dns and than issue these two commands:

mysql -u root -p<password> < vpslinkdb.sql
tar xvf vpslinkbackup.tar.gz

Make sure your configuration is setup in your new host like the http configuration which you may need to restart.  Other than that because you had the foresight to have a folder structure and database structure.  This did not impact all all my existing domains at enotch.  Hope this helps someone.

Here’s a link to the enotch vps performance stats forums/topic.php?id=4. I have to mention that I would not recommend an openvz vps for business sites.

#UPDATE May 25, 2010 – I was updated and no problems at all but I only have one domain in vpslink but the monitoring service reported no downtime since final migration but than I am not honestly hosting any active site.  I still recommend vpslink but with a few hesitations.  They changed the processor to xeon instead of a core2duo which is faster but removed one cpu core.  So now I have one xeon core instead of 2 core 2 duos.  If you order it now you will be in the boston datacenter but everything is working well as far as I can tell.
