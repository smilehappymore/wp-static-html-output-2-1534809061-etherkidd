Cloudflare review

Jan 25th, 2011
Leave a comment

I used cloudflare for around 2 months on and continue to use cloudflare for my very spotty reseller shared hosting account with good results and so I will detail why you may or may not want to use cloudflare with your domain and some tips that may be useful.  I only used the free version and cannot comment on the advance security features and other paid features like preloading.

If you didn’t know already; cloudflare is a free content delivery network that acts as a proxy between you and your visitors.  This is accomplished by setting your nameservers for your domain to those designated to you when you sign up for cloudflare  This takes less than 5 minutes but require you to check the dns entries cloudflare was able to detect for your domain.  They particular had difficulty in subdomains and any future subdomains or domains you add in your shared hosting provider you also need to add into cloudflare’s dns interface to work properly.  On the other hand, cloudflare’s dns implementation is AWESOME.  It has the most features I’ve seen from any free dns provider and most likely superior than what is available in a shared hosting environment.  For example they allow you to add a LOC record.  That is completely mental and redonkulous.

Cloudflare’s caching feature is pretty good but like all caching mechanisms; takes a bit more care and understanding as it can cause problems which can be difficult to diagnose.   For example whenever you want to edit the rewrite rules in apache’s .htaccess you should disable cloudflare temporarily by going into development mode because cloudflare actually caches rewrite redirects so you may be confused why your .htaccess changes are not working properly.  The caching service may also experience problems which can cause slower speeds and other misc errors which may require you to disable it in the cloudflare web interface.  They even cache pages for offline browsing but seem to work only on the front page and rarely on other pages.  This takes 3 days or more.

Because cloudflare can act as a proxy it can protect you from spam, bots, and some internet threats.  I know this because my modest domain which receives around 600 spam messages a month changed to a very modest 30ish.  The security features is a tempting feature for those who have a blog and forum but probably meaningless to a static site.  However you can turn off/on cloudflare by domains and subdomains so some planning is a good idea.  Another reason you may wish to turn off cloudflare is if you like better statistics since cloudflare blocks dangerous elements from accessing your site.

Some interesting quirks are one, they monitor outgoing links using viglink.  This adds viglink’s script on every page which can affiliate links to some websites you have not affiliated.  This is not bad if you are using the free version and want to thank them for service but may not be so good if your site is based on reviews as it may be a conflict of interest.  The second issue is cloudflare’s statistics; which may not always be available at times.  You should also consider their page load estimates to be a bit off but that is true of all page load tests.  The third issue really is for me their site down messages which does not auto refresh the page as it gets a bit annoying when your hosting provider is experiencing issues like high iowait but should still be able to display a page but shows instead as a site down message.

Bottom line:  You really can’t go wrong in setting up the free cloudflare version on any shared hosting account which includes a vps where you are experiencing load problems.  Cloudflare is very efficient in managing static files which is why their caching will go a long way in a shared hosting account when apache is known to perform poorly in serving such files.  This becomes less important in a vps as you can install nginx(cloudfare uses nginx) or some other low resource static content server to mitigate apache resources.

Worth the price? Yes but for 20 dollars a month you can move out of shared hosting to a reliable vps provider as listed here (my xen vps providers).  If you already have an vps account… for 20 dollars you can mirror your existing vps.  I can see this as appealing for those who have a managed vps as it is probably cost effective and provide additional benefits that make it worthwhile than upgrading their existing managed vps plan.  If you are extremely large with your own server and serving globally via your website,  it is probably more cost effective than other cdns but requires additional tweaking on your part if it is video heavy by direct streaming as it will not serve properly in a cloudflare enabled domain/subdomain.  This may not even be an option since some large websites probably employ a third party dns service.

Caveats: If your geographic audience is closer to your web server than cloudflare’s server locations than you may want to skip it.  On a shared hosting plan you will need the providers cooperation in setting apache/litespeed/etc to provide accurate web statistics on awstats/etc.  Since cloudflare caches static content; if you use some sort of script that autoupdates one image every 5 minutes; that image will be cached longer by cloudflare. HTML doesn’t count as a static file so it’s not going to help when your server is experiencing problems except their site offline notice.

Room for improvement:  The most annoying aspect of cloudflare is their off line browsing page.  I think it should be customized so their branding is not there or the ability to serve select custom pages for downtime.  If this is part of their advance feature that is upcoming than it is well worth a monthly paid fee.  I can also see them offering a service to compete with dnsmadeeasy since they already have some of the infrastructure in place.

Tips:  1) setting subdomains for parallel content distribution is easy with cloudflare and works great for better performance.  2) You can enable google analytics from cloudflare’s settings which allows you to remove the code from your source.

Misc:  Cloudflare support is amazing.  I want to thank Damon Billian for the cloudflare tshirt when I sent a support email.  The tshirt was awesome but I don’t know how cloudflare was able to send it out by usps so fast.  Basically I told him the size after 3pm in california and he must have sent it on the same day Wednesday but it arrived the day after on a Friday.  2 business days from California by usps to New York is crazy.
