ezpzhosting.co.uk budget vps review

Jan 30th, 2012
Leave a comment

So I have not been happy with ezpzhosting.co.uk reseller hosting for a while; a recent uptime notification spam in January made me request a credit so I can use some other service that I might be happy with.  Fortunately Dan at ezpzhosting.co.uk was understanding and allow that to happen.  So this is my review which is somewhat favorable especially since the performance of my wordpress on the vps is much more responsive.  So here’s a quick list of the pros and cons of my budget vps package in the united kingdoms in the city of Kent (datacenter) named u-vps-128. 128mb ram with burst up to 256mb:

Pros:

  • 500gb bandwidth.  very generous
  • £4.99 equals to around $7.65 us dollars.  This will fluctuate higher or lower with currency.  A good price considering the bandwidth.
  • There are discounts for prepaying ahead of 6 months and a year but has a no refund policy; so be cautious and use your own judgement.
  • SolusVM control panel management.  has serial console so you can access it even if ssh is not installed or failed.

Cons:

  • only openvz available.  xen would be preferable
  • Uses older kernel. uname produces 2.6.18-238.9.1.el5.028stab089.1 in debian
  • no ipv6 available
  • reverse dns requires a ticket and currently disabled in soluxvm
  • Access to only 1 cpu core at 2327.496hz. Actually not a bad thing.

Troubleshooting: I did run into several problems with my vps which leads me to believe they are lacking a bit of experience in the field; I’ll detail some of the issues I’ve experienced in no particular order.  First thing to get out of the way is, I seemed to had purchased the vps when they were planning to move their datacenter.  Originally the vps was in Manchester and it has now moved to Kent.  This is not a big deal but this usually only happens to smaller vps providers.  The move caused an ip to be issued that was different than the one I had, but data remained the same.  The second issue was the solusvm control panel showed a package bandwidth limitset at 100gb instead of the 500gb advertised.  This was quickly fixed with a support ticket.  The third issue is the vps templates was not tested properly.  For example debian 5 did not have locales installed so I could not install the linux benchmark program properly.  The iptables modules like recent was not working so I could not secure the vps a little.  In debian 5 the default sources list was not correct so I had to edit the file to do an update.  Aptitude was not installed in debian 5 so that had to be installed.  Updates was still not working because the default templates had an init file “/etc/init.d/vzquota” which did not conform to standard debian init.d script requirement.  The file /etc/init.d/vzquota will also get rewritten back to default on reboot so an init.d script had to be written to append the “/etc/init.d/vzquota” to correct the issue.  Everything however is now working properly and I’m happy with the vps.  This review does not include any network or hardware issues as I’ve not encountered any yet however I will update it as appropriate, when it happens.  So far though I would recommend the vps if you are looking for large bandwidth in the United Kingdom as the price is very competitive.

Recommended:  I use debian squeeze because it is easier to configure for a lower memory footprint.  However ezpzhosting.co.uk’s debian 6 squeeze minimal template includes rsyslogd which allocates a large amount of memory in privvmpages on an openvz system.  This caused my mysql to partially fail installation so I recommend you remove it and install something like inetutils-syslogd.  However the vps uses an old kernel that is more suited for Centos.  You will probably experience fewer problems using Centos.
