WordPress Troubleshooting

Dec 5th, 2009
Leave a comment | Trackback

I recently had a problem with wordpress where the whole front page was a blank page. I solved the problem so I decided to write this post on solving the problems I encountered. The problem occured after a reboot command to my vps after I made some changes and I wanted to see if a reboot will bring all the services back up working. That is when I noticed was nothing but a blank page. I am not exactly sure what happened however the administrative login was available but the theme of the login page appeared broken (first clue). At first I thought my whole database was gone however it appeared the opcode cache was to blame and when I cleared it the backend showed all the data available but the site still was a blank page.

1) Solution: the theme was broken. Go back to the appearance tab and change the theme to another one and than back again to see if that fixes the issue (it did for me)

Problem 2: All my settings was reset. (including permalinks)
2) Solution: Go back to all the settings tab and re-enter all the information. This is tedious but good time to check everything.

Problem 3: All plugins was disabled.
3) Solution: Great time to reactive the plugins and see if you really need them.

Problem 4: Akismet Configuration showed: “The key you entered is invalid. Please double-check it.”
4) Solution: Apparently akismet passes the url in the general settings that was reset to blank. Readd your url in the “wordpress url” to correct the issue.

Everything should work as normal at this point however here’s a few interesting things that might be of interest:
+ check for core. files in your wordpress directory. These are core dumps for when i guess wordpress related crashes(like above). My files was 150M so check yours and delete them at your leisure.  Crashes can be caused by plugins so disable only the nonessential ones and slowly add them over a 2-3 weeks to see if there are any core files in your system.
+ akismet spam count increases but nothing is in your spam queue – I think this is because you might be using akismet with bbpress or you have “Automatically discard spam comments on posts older than a month.” checked.  You can basically ignore it but check if it’s working but making a comment.  If you see it listed than it’s working as should.  You should check the users and delete the spam ones.
+ irrelevent incoming links – this also has to do with the “incoming links” being reset to default “http://blogsearch.google.com/blogsearch_feeds?hl=en&scoring=d&ie=utf-8&num=20&output=rss&partner=wordpress&q=link:”. Just add your domain in the end like “”
+strange characters in old posts – This is a weird one. This link suggests commenting out define(‘DB_CHARSET’, ‘utf8’); and define(‘DB_COLLATE’, ”); in wp-config.php.  However this just lead to � in the spaces.  The more accurate solution however seems to be from this thread.  The problem was during my blank page reset it also cleared my settings->reading->Encoding for pages and feeds.  The default was utf-8 which is usually what you have on your wp-config since 2.8, however your setting might be different. However you should try the rest of the solutions in that thread as it is very thorough.

Some helpful tips below:

Post revisions – If it’s a personal blog you might not need to keep a long list of revisions.  The details on setting it is listed here.  I personally set it at define('WP_POST_REVISIONS','3');

For errors such as:

Database Upgrade Required

WordPress has been updated! Before we send you on your way, we have to upgrade your database to the newest version.

The upgrade process may take a little while, so please be patient.

Upgrade WordPress Database

You can follow this link as discussed here.  Basically you should match db_version in database (inside WP_Options table)  with wp-include/version.php line of $wp_db_version = 12329; (for version 2.9).  However if you find your wordpress blog working fine except for accessing the wp-admin interface; the answer is actually to flush your cache (object cache or memcache for example).  Very tricky one but if you like you can restart memcache or reboot your server which would basically do the same thing.
